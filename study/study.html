Variable

1. JavaScript primitive (원시값)코드보기

1) 객체가 아니면서 메소드(method)와 속성(property)을 가지지 않는 데이터

2) 종류

  1. string :   문자 타입
  2. number :   숫자 타입
  3. boolean :   true(데이터가 있으면 true로 간주 - 1, 1.5, 문자열) / false(데이터가 없으면 false로 간주 - 0, 0.0, "", undefined, null)
  4. undefined :   선언 후 값을 할당하지 않은 변수 또는 값이 전달되지 않은 매개변수에 자동으로 할당되는 값
  5. null :   존재하지 않거나 유효하지 않은 object 또는 주소를 의도적으로 가리키는 참조

                                    
                                        // typeof - 값의 데이터 타입 을 확인할 때 사용
                                        console.log(typeof "Hello");        // string(문자열)
                                        console.log(typeof "1");            // string(문자열 - ""을 사용하였기 떄문에 숫자 X)
                                        console.log(typeof 1);              // number(숫자)
                                        console.log(typeof true);           // boolean
                                        console.log(typeof false);          // boolean
                                        console.log(typeof undefined);    // undefined
                                        console.log(typeof null);           // null

2. JavaScript 변수 선언코드보기

스코프 재선언 할당 전 재할당 용도
var 전역 스코프 가능 undefined 가능 전역 변수
let 블록 스코프 불가능 undefined 가능 지역 변수
const 블록 스코프 불가능 사용 불가능 불가능 상수 
                                    
                                        // 코드 블록 구성하기 : { ... }
                                        {
                                            // 변수 a 선언
                                            var a;
                                            console.log(a); // undefined (var의 값을 할당하기 전에는 undefined)

                                            // 변수 a 재선언 (var 가능)
                                            var a;

                                            // 값 할당
                                            a = 10;
                                            console.log(a);  // 10
                                            a = 20
                                            console.log(a);  // 20   // 재할당 가능
                                        }
                                        console.log(a);  // 20 (var는 전역 스코프 이므로 코드 블록 밖에서도 확인이 가능)

                                        {
                                            // 변수 x 선언
                                            let x;
                                            console.log(x); // undefined (let의 값을 할당하기 전에는 undefined)

                                            let x;  // 재선언 불가능

                                            x = 2;  // 값 할당
                                            console.log(x)  // 2
                                            x = 4;
                                            console.log(x)  // 4 (재할당 가능)
                                        }
                                        console.log(x); // let은 블록 스코프 이므로 코드 블록 밖에서 확인이 불가능

                                        {
                                            // const 변수는 반드시 선언과 할당을 모두 해야 함 (초기값이 반드시 필요)
                                            const y;    // 값이 할당되지 않았기 때문에 할당 필요
                                            const y = "hello world";    // 값을 할당해야만 한다

                                            y = "hello World";  // 재할당 불가능
                                        }
                                        console.log(y); // const는 블록 스코프 이므로 코드 블록 밖에서 확인이 불가능

3. JavaScript 변수명(식별자) 규칙코드보기

1) 문자, 숫자, 밑줄(_), 달러($) 기호 사뇽 가능

2) 첫 글자는 숫자일 수 없다

3) 대소문자를 구분한다

4) Camel Case 방식의 이름 규칙을 사용

                                    
                                        let hello;          // 문자로 시작하는 변수명
                                        let hello2;         // 문자 + 숫자
                                        let _hello;         // 밑줄(_)fh tlwkr
                                        let HELLO;        // 대문자 (대소문자 구분)
                                        let helloWorld;     // camel case (첫 단어를 제외 두 번째 단어의 첫 글자는 대문자 사용)
                                        let $hello;         // $기호로 시작
                                        let 2hello;         // 숫자로 시작(불가능)

                                        /*
                                        이름 규칙의 종류
                                        1) camelCase (카멜 케이스)
                                            - firstName, totalCount, isLoggedIn
                                            - 첫 단어는 소문자, 다음 단어는 첫 글자 대문자
                                            
                                        2) PascalCase (파스칼 케이스)
                                            - FirstName, TotalCount, IsLoggedIn
                                            - 모든 단어의 첫 글자를 대문자

                                        3) snake_case (스네이크 케이스)
                                            - first_name, total_count
                                            - 소문자 + 밑줄(_) 로 구분

                                        4) kebab-case (케밥 케이스)
                                            - first-name, total-count
                                            - 소문자 + 하이픈(-) 으로 구분
                                            - 자바스크립트에서는 사용 불가 (하이픈이 연산자로 인식됨)
                                        */

4. Template literals코드보기

1) 백틱(``)으로 묶은 문자열을 의미

2) 여러 줄로 이루어진 문자열을 처리할 수 있다

3) 문자열 보간 기능을 사용할 수 있다

                                    
                                        // 다중 라인 처리
                                        const iconArt = `^      ^
                                                                                --              `

                                        // 보간 기능 - 문자열 안에 변수나 표현식을 ${...} 형태로 삽입 가능
                                        // 1 + 2 는 3 이다.
                                        const a = 1;
                                        const b = 2;
                                        const c = 3;
                                        console.log(a + " + " + b + " 는 " + c + " 이다.");     // 기존 문자열 연결법
                                        console.log(`${a} + ${b} 는 ${c} 이다.`);               // 리터럴 방식

5. 호이스팅(hoisting) ★★★★★코드보기

1) 끌어 올리다

2) javascript 인터프리터가 코드를 실행하기 전에 var 변수 선언, 함수 선언 또는 import를 해당 범위의 맨 위로 끌어올리는 것처럼 보이는 현상

3) var 키워드로 선언된 변수는 코드 어디에 존재하든 가장 먼저 처리되는 것으로 이해할 수 있다.

4) 변수 선언과 초기화가 동시에 진행되는 경우 변수 선언만 분리해서 먼저 처리

                                    
                                        // 변수 선언을 하지 않고 바로 사용 가능(var 변수만)
                                        money = 50000;
                                        console.log(money); // 50000
                                        var money;  // var 변수 선언은 호이스팅에 의해서 항상 먼저 처리

                                        console.log(cat);   // undefined (변수 cat 은 호이스팅 되었지만 값은 해당되지 않음 - 선언만 호이스팅)
                                        var cat = "고양이";

Operator

1. 산술 연산자코드보기

1) 더하기 : +

2) 빼기     : -

3) 곱하기 : *

4) 나누기 : /

5) 나머지 : %

                                    
                                        const a = 5;
                                        const b = 2;

                                        console.log(a + b); // 5 + 2 = 7
                                        console.log(a - b); // 5 - 2 = 3
                                        console.log(a * b); // 5 * 2 = 10
                                        console.log(a / b); // 5 / 2 = 2.5
                                        console.log(a % b); // 5 % 2 = 1
                                        console.log(parseInt(a / b));   // 2(나눗셈의 몫)

2. 증감 연산자코드보기

1. ++a : 전위 증가 (증가 후에 사용)

2. a++ : 후위 증가 (사용 후에 증가)

3. --a : 전위 감소

4. a-- : 후위 감소

BOM

1. window

1) 현재 스크립트가 작동 중인 창을 나타내는 전역 객체

2) 모든 JavaScript 객체의 최상위 객체

3) JavaScript 코드상에서 생략 가능 (window.alert() > alert())

2. history코드보기

1) 방문했던 경로를 기억하고 있는 객체

2) 히스토리 객체를 이용해서 이전 화면으로 이동 가능

3) 사용예시(코드확인)

                                    
                                        1) 뒤로 가기        : history.back() == history.go(-1)
                                        2) 앞으로 가기      : history.forward() == history.go(1)
                                        3) 특정 위치로 가기 : history.go(N) / N이 음수면 뒤로 N 만큼, N이 양수면 앞으로 N 만큼

3. location코드보기

1) 문서의 현재 위치에 대한 정보가 담긴 location 객체를 반환

2) 새로운 페이지 이동 / 새로고침

                                    
                                        location.href = "https://www.naver.com"     // 뒤로가기 가능
                                        location = "https://www.naver.com"          // href 생략 가능
                                        location.assign("https://www.naver.com")    // 뒤로가기 가능
                                        location.replace("https://www.naver.com")   // 뒤로가기 불가능
                                        location.reload()                           // 새로고침(캐시사용)
                                        location.reload(true)                       // 새로고침(캐시사용안함)

4. screen

1) 브라우저 화면 정보를 관리하는 객체

2) 화면의 너비/높이 정보를 알아낼 수 있다.
